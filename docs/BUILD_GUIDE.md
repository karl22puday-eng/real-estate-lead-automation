# Build Guide ‚Äî Real Estate Lead Automation & CRM System

A step-by-step to stand this up on a 100% free stack: **n8n Cloud ¬∑ Supabase ¬∑ Groq ¬∑
Gemini ¬∑ Telegram ¬∑ GitHub Pages**, with optional **Meta Lead Ads**.

---

## 0. Accounts you'll need (all free)
- **Supabase** ‚Äî <https://supabase.com> (Postgres + pgvector)
- **Groq** ‚Äî <https://console.groq.com> (Llama 3.3 70B, reasoning)
- **Google AI Studio** ‚Äî <https://aistudio.google.com> (Gemini embeddings)
- **Telegram** + **@BotFather** (hot-lead alerts)
- **n8n Cloud** ‚Äî <https://n8n.io> (free trial; or self-host with Docker)
- **GitHub** (Pages hosting for the form + dashboard)
- *(optional)* **Meta for Developers** ‚Äî <https://developers.facebook.com> (Lead Ads)

## 1. Supabase (database + vector store)
1. Create a new project. Note the **Project URL** and, under *Settings ‚Üí API*, the
   **anon** key (public) and **service_role** key (secret).
2. Open *SQL Editor ‚Üí New query*, paste all of [`db/schema.sql`](../db/schema.sql), Run.
   This enables `pgvector` and creates `leads`, `kb_documents`, the `match_documents` and
   `lead_exists` RPCs, RLS, and the sanitized `leads_public` view.

## 2. Collect keys into `.env`
Copy [`.env.example`](../.env.example) ‚Üí `.env` and fill in everything. (`.env` is
gitignored ‚Äî it's your local reference; the real secrets live in n8n credentials.)

## 3. Telegram bot
1. Message **@BotFather** ‚Üí `/newbot` ‚Üí copy the **bot token**.
2. Message your new bot once, then visit
   `https://api.telegram.org/bot<token>/getUpdates` and copy your numeric **chat id**
   (this is the chat that receives hot-lead alerts).

## 4. Generate the workflows from source
The workflow JSON is generated by scripts so it's reproducible and reviewable.
1. Open both files in [`scripts/`](../scripts/) and set the placeholders at the top:
   - `$SUPA` ‚Üí your Supabase project URL (both scripts)
   - `$TG_CHAT` ‚Üí your Telegram chat id (qualification script)
2. Run them (PowerShell):
   ```powershell
   ./scripts/build-ingestion-workflow.ps1
   ./scripts/build-qualification-workflow.ps1
   ```
   Each prints `Valid JSON. Nodes: N`.

## 5. Import into n8n + attach credentials
1. In n8n: *Workflows ‚Üí Import from File* ‚Üí import both files from [`workflows/`](../workflows/).
2. Create credentials and attach them to the HTTP/Telegram nodes:
   - **Supabase API** (used by the Supabase HTTP nodes) ‚Äî project URL + service_role key.
   - **Header Auth ‚Äî Gemini:** header `x-goog-api-key` = your Gemini key (on the
     *Embed* nodes). *(Or use `Authorization: Bearer` per the Gemini docs.)*
   - **Header Auth ‚Äî Groq:** header `Authorization` = `Bearer <GROQ_API_KEY>` (on *Groq Agent*).
   - **Telegram** ‚Äî your bot token.
3. **Run `01 KB Ingestion`** once (Manual Trigger) to chunk + embed the `kb/` docs into
   `kb_documents`. Confirm rows appear in Supabase.
4. **Activate `02 Lead Qualification`** and copy its **production webhook URL**
   (`‚Ä¶/webhook/realestate-intake`).

## 6. Frontend (GitHub Pages)
1. In [`frontend/index.html`](../frontend/index.html), set `WEBHOOK` to your production
   webhook URL.
2. In [`frontend/dashboard.html`](../frontend/dashboard.html), set `SUPABASE_URL` and
   `ANON_KEY` to your project's values (anon key is safe here ‚Äî RLS limits it to the
   sanitized view).
3. Push to GitHub; in *Settings ‚Üí Pages*, set **Source: GitHub Actions**. The included
   workflow deploys `/frontend` on every push.

## 7. Test (do all of these before calling it done)
- **Happy path / hot lead:** submit as a pre-approved buyer, ASAP, in a covered area ‚Üí
  expect a high score, a row in `leads`, a Telegram alert, and a grounded reply.
- **Cold lead:** "just browsing / just researching" ‚Üí captured, low score, no alert.
- **Duplicate:** submit the same email + message twice ‚Üí second returns instantly as a
  duplicate, no second row, no second alert.
- **Bad input:** missing email or intent ‚Üí HTTP 400 with a clear error, nothing stored.
- **Dashboard:** confirm leads appear sorted by score and that no email/phone is exposed
  (check the network response ‚Äî it should only contain `leads_public` fields).

## 8. (Optional) Meta Lead Ads
Follow [`META_LEAD_ADS.md`](META_LEAD_ADS.md) to feed Facebook/Instagram lead ads into the
same pipeline (native n8n trigger or Meta webhook + Graph API), mapping fields to the
intake shape so validation/scoring/routing work unchanged.
